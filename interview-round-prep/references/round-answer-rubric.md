# Round Answer Rubric

用于判断多轮面试准备文档是否真的可用。

## 必须达标

| 指标 | 达标标准 |
|-|-|
| 轮次区分 | 一面、二面、三面/终面、HR 面的考察重点明显不同 |
| JD 贴合 | 每轮题目都能回到 JD 的职责、能力要求或隐含考点 |
| 直接可用 | 每轮至少 3 道核心题有 60-120 秒可直接朗读答案 |
| 案例连续 | 核心答案尽量围绕 1-2 个统一项目故事展开 |
| 数据密度 | 每段直接答案至少 2 个指标或清晰的指标占位 |
| 来源标注 | 题目标注真题参考 / 相似题迁移 / JD 推演题 |
| 替换友好 | 标出用户必须替换的项目、指标、角色、公司、时间 |
| 风险控制 | HR 面和终面必须覆盖离职原因、动机、稳定性、薪资和岗位匹配风险 |

## 好答案标准

- 能解释这一轮面试官为什么问这道题。
- 能告诉用户该题应该用哪个项目故事回答。
- 能给出一段用户马上可以开口练的答案。
- 有业务细节和指标口径，不只说“提升体验”。
- 能给同一道题在不同轮次的表达差异。
- 有追问预判，但不把文档主题写成“追问训练”。

## 常见失败

- 只把同一批题重复分到不同轮次。
- 一面和二面答案没有差异，都是项目流水账。
- HR 面只给模板话术，没有解释风险和边界。
- 答案过度虚构，用户无法替换成自己的真实经历。
- 只有回答思路，没有可直接复制练习的答案。
- 多个答案里的指标和项目背景互相冲突。
