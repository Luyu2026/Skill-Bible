# interview-round-prep

面试全流程准备生成器。

它适合用在这样的场景：你已经有一个目标岗位 JD，或者已经用 `interview-prep-brief` 生成过面试备书，接下来想按一面、二面、三面/终面、HR 面分别准备高概率题和可直接开口答案。

## 它解决什么问题

很多面试准备只会给一堆题，但真实面试不是“一轮题库”，而是一场多轮筛选。不同轮次的面试官关心点不一样，同一道题在一面、二面、终面和 HR 面的回答重心也不一样。

这个 Skill 更强调三件事：

1. 把岗位面试拆成多轮作战书，而不是混在一起准备。
2. 按面试官身份判断每轮真正想验证什么。
3. 每轮都给出高概率题、回答思路、可直接开口答案和追问预判。

## 一个重要机制

它默认优先使用用户提供的 JD、简历、项目经历、题库/面经，以及 `interview-prep-brief` 已经生成的面试备书。

如果材料不足以支撑高质量拆解，或者用户明确要求进一步调研，Skill 可以参考 GitHub 上的 mock interview / interview prep / question generator / interview coach 项目。但它参考的不是题目内容，而是产品机制，例如：

- 如何按岗位和 JD 生成问题。
- 如何区分一面、二面、终面、HR 面。
- 如何做连续追问。
- 如何给回答反馈和改进建议。
- 如何把弱点沉淀成练习计划。

换句话说，外部参考是一个“多轮拆解能力兜底机制”：让面试准备更像真实流程，而不是去网上照搬题目。

## 输出内容

- 先看结论：这个岗位多轮面试最可能考什么。
- 岗位能力拆解：4-6 个核心能力维度。
- 贯穿全场的项目故事：2-3 个适合反复回答的项目。
- 一面准备：证明你真的做过，包含专业基本功和项目细节。
- 二面准备：证明你会判断和推进，包含业务判断、指标体系和跨团队协作。
- 三面/终面准备：证明你有方向感，包含战略理解、owner 意识和长期潜力。
- HR 面准备：降低用人风险，覆盖动机、稳定性、离职原因和薪资预期。
- 最后一周练习计划：把题目练习拆成可执行动作。

## 轮次拆解原则

多轮面试不是简单地把题目平均分配到每一轮，而是按面试官身份和风险点重排。

| 轮次 | 面试官常见身份 | 核心考察 | 回答重心 |
|---|---|---|---|
| 一面 | 直属 leader / 资深同岗位 | 经历真实性、专业基本功、项目细节 | 讲清做过什么、怎么做、数据结果、个人贡献 |
| 二面 | 业务负责人 / 跨团队负责人 | 业务判断、方法论、复杂问题推进 | 讲目标拆解、取舍逻辑、指标体系、协同机制 |
| 三面/终面 | 部门负责人 / 高层 | 战略匹配、owner 意识、潜力和边界 | 讲长期判断、资源约束下的选择、对业务的理解 |
| HR 面 | HRBP / 招聘负责人 | 动机、稳定性、薪资、风险和文化匹配 | 真诚解释选择逻辑，控制风险表达，避免过度包装 |

同一道题在不同轮次要调整重心。比如“讲一个 AI 产品项目”：一面讲细节和个人动作，二面讲判断和协作，终面讲业务理解，HR 面讲职业选择。

## 输出方式

优先支持飞书文档；如果当前环境没有配置 `lark-cli`、没有授权或没有飞书权限，则自动生成本地 Markdown 文档。

推荐默认输出：

- 有飞书：`<公司><岗位> 多轮面试准备` 飞书文档。
- 无飞书：`interview-round-prep/<公司或岗位>-多轮面试准备.md`。

## 第一次使用

你只需要给几类信息，越完整效果越好：

1. 岗位 JD：截图、链接、复制文本都可以。
2. 个人背景：简历、项目经历、自我介绍或几段关键经历。
3. 已有备书：如果已经用 `interview-prep-brief` 生成过，可以直接贴飞书链接或 Markdown。
4. 面试流程：一面/二面/终面/HR 面，未知也可以。
5. 输出偏好：飞书文档、本地 Markdown，或先在对话里预览。

如果你只有 JD，Skill 也可以先生成一版多轮面试准备；但如果要生成可直接开口答案，最好补充自己的真实项目和数据。

## 推荐输入模板

```text
请使用 interview-round-prep，基于下面的 JD 和我的经历，生成一份多轮面试准备文档。

岗位/公司：
JD：
我的经历：
已有面试备书：有/没有，链接或正文如下
面试流程：一面/二面/终面/HR 面/未知
输出方式：飞书文档 / 本地 Markdown / 先预览

要求：
1. 按一面、二面、三面/终面、HR 面拆分
2. 每轮都给高概率题、考察点、回答思路、可直接开口答案和追问预判
3. 答案尽量围绕同一个项目故事，不要每题换一个故事
4. 不要输出替换字段或替换清单，直接写成可练习版本
```

## 直接答案要求

可直接开口答案不能只讲方法论，必须包含：

- 一个能贯穿多轮的项目故事。
- 2-4 个产品/业务指标。
- 明确的个人动作和贡献。
- 面试官可能继续追问的方向。
- 不同轮次下的表达重心差异。

示例指标包括：CTR、转化率、ROI、回答成功率、满意度、负反馈率、留存率、复访率、GMV、GPM、人工兜底率。

如果用户已经提供真实经历，答案应直接基于这些经历写，不要再输出 `替换字段`、`待替换` 或 `示例可替换`。

## 适合谁

- 校招/社招求职者：已经拿到面试机会，需要按轮次集中准备。
- 面试前一周冲刺的人：希望把题目、答案和追问压缩成一份作战书。
- 已有面试备书的人：想继续拆成一面、二面、终面和 HR 面的练习材料。
- 求职辅导者：需要为学员生成更贴近真实面试流程的训练文档。
