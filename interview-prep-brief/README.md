# interview-prep-brief

岗位面试备书生成器。

它适合用在这样的场景：你拿到一个陌生岗位 JD，想快速知道这个岗位大概率会问什么、该怎么准备、哪些回答可以直接开口练。

## 它解决什么问题

普通 AI 很容易生成一份“可能面试题 + 标准答案”，但这和通用提示词差别不大。这个 Skill 更强调三件事：

1. 从 JD 反推岗位能力模型，而不是泛泛列题。
2. 参考已有题库/面经的考察思路，但不把所有推演题都伪装成真题。
3. 给出带业务细节和数据的可朗读答案，让用户能马上开始练。

## 输出内容

- 岗位速读：一句话说明岗位真正考什么。
- 能力模型：拆出 4-6 个核心能力维度。
- 高概率题目地图：标注 `真题参考`、`相似题迁移`、`JD 推演题`。
- 答案骨架：告诉用户如何替换成自己的经历。
- 直接可用参考答案：带数据、带业务细节、可直接朗读。
- 追问链路：模拟面试官连续追问。
- 评分标准：好答案、普通答案、危险答案的区别。
- 个人素材缺口：用户还需要补哪些项目、数据、案例。
- 3 天备面计划：把准备动作拆到每天。

## 题目来源原则

题库不是唯一来源。已有面经和知识库主要提供三种价值：

- `题目参考`：题库里出现过的真实问题。
- `答题思路参考`：题库答案里的拆解方式、指标口径、表达结构。
- `岗位考察点参考`：同类岗位反复出现的能力要求。

如果某道题不是题库原题，但能从 JD 和同类问题推导出来，应标注为 `JD 推演题` 或 `相似题迁移`，不要硬说是真题。

## 默认参考库与权限

作者维护了一份私域题库《大厂面试真题解析》，可作为默认参考：

https://gte09oerz5.feishu.cn/wiki/E5UxwObpMiwyfVk6j0BcsMS7nGc

如果你没有访问权限，Skill 会自动跳过，不影响使用。你也可以提供自己的题库、面经、飞书文档、Notion、Markdown 或截图。

这份默认题库只用于参考题型、答题思路和岗位考察点，不代表每道生成题都是真题。

## 输出方式

优先支持飞书文档；如果当前环境没有配置 `lark-cli`、没有授权或没有飞书权限，则自动生成本地 Markdown 文档。

推荐默认输出：

- 有飞书：`<公司><岗位> 面试备书` 飞书文档。
- 无飞书：`interview-briefs/<公司或岗位>-面试备书.md`。

## 第一次使用

你只需要给三类信息，越完整效果越好：

1. 岗位 JD：截图、链接、复制文本都可以。
2. 个人背景：简历、项目经历、自我介绍或几段关键经历。
3. 输出偏好：飞书文档、本地 Markdown，或先在对话里预览。

如果你没有题库、没有飞书、也没有完整简历，Skill 仍然可以先基于 JD 生成一版备书；缺失的经历和数据会用 `[你的真实项目]`、`[你的真实指标]` 标注，方便后续替换。

## 推荐输入模板

```text
请使用 interview-prep-brief，基于下面的 JD 和我的经历生成一份面试备书。

岗位/公司：
JD：
我的经历：
面试阶段：一面/二面/终面/未知
输出方式：飞书文档 / 本地 Markdown / 先预览

要求：
1. 标注题目来源：真题参考 / 相似题迁移 / JD 推演题
2. 给出带数据的可直接开口练答案
3. 模拟 3 轮面试官连续追问
4. 列出我还需要补的项目素材和指标
```

## 直接答案要求

可直接用的答案不能只讲方法论，必须包含：

- 一个统一的项目故事。
- 2-4 个产品/业务指标。
- STAR 或类似结构。
- 可替换的个人素材位置。
- 面试官可能继续追问的方向。

示例指标包括：CTR、转化率、完播率、ROI、回答成功率、满意度、负反馈率、人工兜底率、次日复访率。

## 适合谁

- 秋招/社招求职者：面对陌生岗位，快速形成备面材料。
- 求职辅导者：把 JD、真题、答案、追问、评分标准串成一份交付物。
- 内容创作者：把面试题拆解成可讲解、可复盘的内容。
