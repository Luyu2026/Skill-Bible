# interview-prep-brief

岗位面试备书生成器。

它适合用在这样的场景：你拿到一个陌生岗位 JD，想快速知道这个岗位大概率会问什么、该怎么准备、哪些回答可以直接开口练。

## 它解决什么问题

普通 AI 很容易生成一份“可能面试题 + 标准答案”，但这和通用提示词差别不大。这个 Skill 更强调三件事：

1. 从 JD 反推岗位能力模型，而不是泛泛列题。
2. 参考已有题库/面经的考察思路，但不把所有推演题都伪装成真题。
3. 给出带业务细节和数据的可朗读答案，让用户能马上开始练。

## 输出内容

- 岗位速读：一句话说明岗位真正考什么。
- 能力模型：拆出 4-6 个核心能力维度。
- 高概率题目地图：标注 `真题参考`、`相似题迁移`、`JD 推演题`。
- 答案骨架：告诉用户如何替换成自己的经历。
- 直接可用参考答案：带数据、带业务细节、可直接朗读。
- 追问链路：模拟面试官连续追问。
- 评分标准：好答案、普通答案、危险答案的区别。
- 个人素材缺口：用户还需要补哪些项目、数据、案例。
- 3 天备面计划：把准备动作拆到每天。

## 题目来源原则

题库不是唯一来源。已有面经和知识库主要提供三种价值：

- `题目参考`：题库里出现过的真实问题。
- `答题思路参考`：题库答案里的拆解方式、指标口径、表达结构。
- `岗位考察点参考`：同类岗位反复出现的能力要求。

如果某道题不是题库原题，但能从 JD 和同类问题推导出来，应标注为 `JD 推演题` 或 `相似题迁移`，不要硬说是真题。

## 直接答案要求

可直接用的答案不能只讲方法论，必须包含：

- 一个统一的项目故事。
- 2-4 个产品/业务指标。
- STAR 或类似结构。
- 可替换的个人素材位置。
- 面试官可能继续追问的方向。

示例指标包括：CTR、转化率、完播率、ROI、回答成功率、满意度、负反馈率、人工兜底率、次日复访率。

## 使用示例

```text
请使用 interview-prep-brief，基于下面这个 JD 和我的简历，生成一份面试备书。

岗位：小米 AI 产品经理
JD：……
我的背景：……
可参考知识库：大厂面试真题解析
输出：飞书文档，要求包含可直接开口练的答案和连续追问模拟。
```

## 适合谁

- 秋招/社招求职者：面对陌生岗位，快速形成备面材料。
- 求职辅导者：把 JD、真题、答案、追问、评分标准串成一份交付物。
- 内容创作者：把面试题拆解成可讲解、可复盘的内容。
