# job-application-match

校招/社招投递匹配与岗位版简历生成器。

它适合这样的场景：你有一段真实经历，但不知道哪些岗位值得冲、哪些岗位现在适合投；选完后，还想直接拿到对应岗位能用的简历内容。

## 它怎么工作

1. 先问你是校招还是社招。
2. 校招默认从公开校招岗位池召回候选岗位，也能合并你给的飞书表格、链接、截图或 JD；社招只分析你提供的岗位材料。
3. 先筛毕业届别、学历、城市、截止日期等硬条件，再把“值得冲的好岗位”和“按当前经历适合投的岗位”分开。
4. 对适合投的每个岗位，用表格同时给出真实数据填空版、逻辑自洽示例数据版和使用建议；示例帮助理解完整、有数据的表达标准。
5. 你已有投递清单或收到反馈时，可以直接交给它。它会据此更新岗位优先级、简历版本和下一批投递建议；没有清单也能先跑第一轮。

## 它不会做什么

- 不绕过招聘网站限制，不批量抓取受限职位库，也不自动投递。
- 不把岗位列表页当成完整 JD；只核验少量高相关候选岗位。
- 不把普通参与经历虚构成“主导运营”，不编造数据。
- 不把匹配排序说成 offer 概率。

## 第一次使用

直接把这段发给 Agent 即可：

```text
请使用 job-application-match 帮我找值得投递的岗位，并生成岗位版简历经历。

我是：校招 / 社招
我的经历或简历：
毕业时间、学历、专业（校招必填）：
城市与是否愿意迁居：
目标方向和明确不想投的方向：
我提供的岗位来源（可选：JD、截图、链接、飞书表格）：
我已有的投递清单或反馈（可选：飞书表格、Excel、文档、截图或直接粘贴）：

要求：
1. 校招默认读取 Skill 里的公开校招岗位池；社招只用我提供的岗位材料。
2. 先过硬条件，再做有证据的匹配；岗位链接只核验最相关的几个。
3. 把“值得冲的好岗位”和“按当前经历适合投的岗位”分开；岗位职责单列，薪酬没有可靠来源就写未公开。
4. 对每个适合投的岗位，用表格给我“无真实数据占位版、逻辑自洽示例数据版、使用建议”。
5. 最后按校招或社招给我一份具体的投递行动清单：动作、完成标准、产出物和建议顺序。
6. 如果我给了投递清单或反馈，请直接分析它，并告诉我下一轮要删掉哪些岗位、保留哪些岗位、简历如何调整。
7. 用飞书文档交付；飞书不可用时再给本地 Markdown。
```

## 和其它求职 Skill 的配合

先用它确定值得冲和适合投的方向，并拿到岗位版经历；需要完整简历时再交给 `resume-jd-tailor`。面试准备请单独使用 Skill-Bible 中对应的面试 Skill。

## 迭代机制

用户材料不足以拆清匹配、简历定制或反馈闭环时，Skill 会按需参考 GitHub 上的高质量 job matching、resume tailoring、application tracker 项目，借鉴流程机制而非照搬职位数据或简历内容。
