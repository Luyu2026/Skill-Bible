# Resume Rewrite Method

## 低颗粒经历翻译法

当用户经历很薄，不要直接润色原句。先把经历翻译成岗位能理解的能力证据。

### 第一步：判断可投方向

输入：

```text
我有一段经历：电竞社社员，只参与日常事务，没有正式职位。
我想投互联网岗位，请判断这段经历可以往哪些方向靠，并给匹配分。
```

输出方向示例：

| 方向 | 匹配理由 | 风险 |
|-|-|-|
| 社群运营 | 电竞社天然有社群、成员活跃、规则维护场景 | 需要补社群规模、互动数据、规则动作 |
| 活动运营 | 社团有活动策划、报名、参与、反馈场景 | 需要补活动类型、参与率、复盘动作 |
| 用户运营 | 社员可以被视为用户，有拉新、留存、活跃场景 | 需要补用户分层、调研、留存数据 |
| 内容运营 | 如果做过攻略、资讯、活动预告，可转译为内容供给 | 需要补内容产出频次、阅读/互动效果 |

### 第二步：选目标岗位族

不要一次写所有方向。先选一个岗位族，例如运营，再细分：

- 用户运营
- 社群运营
- 活动运营

### 第三步：把经历拆成四块

每个模块都按同一结构展开：

1. `问题与目标`：当时为什么要做，目标是什么
2. `调研方式`：如何发现问题，如何确认用户需求
3. `具体做法`：从哪些维度拆动作
4. `数据结果`：真实指标；没有真实数据时标注 `[待补充]`，给示例口径

### 第四步：直接给可复制版本

不要让用户在“丰富版/精修版”之间二次判断。最终输出要直接回答两个问题：

1. 可以切入什么岗位。
2. 对应岗位的简历经历应该怎么写。

岗位写法示例：

```text
用户运营：社员留存率 [原始值]，活跃占比 [原始值]，目标提升至留存 [目标值]、活跃 [目标值]。通过线上问卷 + [访谈人数] 名社员访谈，发现 [关键问题]。新增 [活动类型] 满足需求；建立社员档案并通过 [情感连接动作] 增强归属感。[周期] 后，月留存率提升至 [结果值]，活跃占比提升至 [结果值]。
```

直接可复制版本示例：

```text
- 用户运营：社员月留存率 [原始值]，活跃占比 [原始值]，目标提升至留存 [目标值]、活跃 [目标值]。通过线上问卷 + [访谈人数] 名社员访谈，发现活动类型单一问题。新增王者荣耀、CSGO 等活动满足不同成员需求；建立社员档案并通过生日祝福、战绩激励、经验分享会增强情感连接。[周期] 后，月留存率提升至 [结果值]，活跃占比提升至 [结果值]。
```

如果用户给的是京东用户运营这类 JD，优先给：

- 用户运营版本
- 社群运营版本
- 活动运营版本

除非用户明确要求多种排版，不要再拆“丰富版”和“精修版”。

## 表格对比与示例数据

直接可复制区建议用表格承载，避免用户在多段文字里来回找。

表格字段：

| 字段 | 作用 |
|-|-|
| 切入方向 | 用户运营 / 社群运营 / 活动运营等 |
| 无真实数据占位版 | 保留 `[原始值]`、`[目标值]`、`[结果值]`，适合用户填自己的数据 |
| 逻辑自洽示例数据版 | 给一版内部一致、符合场景的数据示例 |
| 使用建议 | 告诉用户投递当前 JD 时优先用哪一版 |

示例数据规则：

- 必须标注为“示例口径，需用户确认后使用”
- 数据要符合小型校园社团规模，不要夸张
- 同一方向里的目标、动作、结果要前后一致
- 不能把示例数据当成用户真实经历

示例：

```text
用户运营示例数据版：社员月留存率约 90%，活跃社员占比不足 50%，目标提升至留存 92%+、活跃 70%+。通过线上问卷 52 份 + 15 名社员访谈，发现活动类型单一、成员连接弱。新增王者荣耀、CSGO 训练/比赛活动，建立社员档案并组织经验分享会。3 个月后，月留存率提升至 92%，活跃占比提升至 75%。
```

## 数据规则

简历要有数据，但不能替用户编数据。

可以输出三种数据表达：

- `真实数据`：用户明确提供，直接使用
- `待补充占位`：`[活动参与率]`、`[社群日均消息数]`、`[留存率]`
- `示例口径`：告诉用户可以怎么量化，例如“从 60% 提升到 80%”“日均消息从 50 条提升到 150 条”

示例口径必须标注为示例，不能写进最终投递版，除非用户确认真实发生。

## 常见经历转译方向

| 原始经历 | 可转译方向 | 证据关键词 |
|-|-|-|
| 社团/学生会 | 用户运营、社群运营、活动运营 | 拉新、留存、活跃、活动参与、用户反馈 |
| 游戏/电竞 | 内容运营、社区运营、用户研究 | 用户心智、社群规则、游戏机制、内容生态 |
| 课程项目 | 产品、数据、策略、市场 | 用户调研、需求分析、方案设计、指标验证 |
| 志愿者/兼职 | 运营、客户成功、销售支持 | 流程执行、用户沟通、服务体验、转化 |
| 自媒体账号 | 内容运营、增长、品牌 | 选题、发布、互动、转化、复盘 |

## 简历 bullet 标准

好 bullet：

- 有目标：为什么做
- 有动作：怎么做
- 有方法：按什么逻辑做
- 有结果：带来什么变化
- 有岗位词：和 JD 能对上

差 bullet：

- “负责日常事务”
- “协助完成活动”
- “提升用户体验”
- “参与社群运营”
- “完成领导安排”

## 面试衔接

简历不是终点。每段改写后的经历都要能回答：

- 这个指标怎么来的？
- 你具体负责哪一块？
- 如果重做一次，你会怎么优化？
- 这个经历和目标岗位有什么关系？
- 哪个动作最关键，为什么？
