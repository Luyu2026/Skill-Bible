# Resume Tailor Evaluation Rubric

判断简历定制是否真的能帮助用户投递。

## 必须达标

| 指标 | 达标标准 |
|-|-|
| JD 贴合度 | 每段核心经历都能回到 JD 的职责或能力要求 |
| 事实边界 | 明确区分用户事实、合理转译、待确认补充、示例口径 |
| 可投递性 | 至少输出 3-5 条可直接放进简历的精修 bullet |
| 关键词覆盖 | 标注 JD 关键词是否已覆盖，指出待补关键词 |
| 数据意识 | 每段经历尽量有指标；无真实数据时保留占位和示例口径 |
| 低颗粒挖掘 | 对薄经历先做方向匹配和经历扩写，不直接空泛润色 |
| 排序建议 | 告诉用户哪些经历应该前置、弱化或删除 |
| 面试可解释 | 改写后的经历经得起面试官追问 |

## 好结果标准

- 能让用户知道“我这段经历到底能投什么”
- 能把普通经历翻译成岗位证据
- 不会为了好看而捏造经历
- 有一版丰富稿帮助用户补细节
- 有一版简历稿可以直接复制
- 有素材缺口，用户知道回去补什么
- 能和后续面试准备 Skill 衔接

## 常见失败

- 只做同义词润色，没有岗位匹配
- 只堆黑话，看不出用户做了什么
- 编造数据，且不标注待确认
- 每条 bullet 都很长，不适合简历
- 没有 JD 关键词覆盖检查
- 把所有经历都写得一样重要，没有排序
- 改完后面试无法解释
