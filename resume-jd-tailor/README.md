# resume-jd-tailor

JD 简历定制改写 Skill。

它适合用在这样的场景：你有一份目标岗位 JD，也有一段原始经历或简历，但不知道怎么把经历写得更像目标岗位需要的人。

## 它解决什么问题

普通 AI 很容易把简历改成一堆漂亮黑话。这个 Skill 更强调三件事：

1. 先从 JD 反推岗位到底要什么能力。
2. 再从用户经历里挖可证明的证据。
3. 最后输出可投递的简历 bullet，同时标注哪些数据需要用户确认。

## 一个重要机制

默认优先使用用户提供的 JD、原始简历、经历片段和简历知识库。

如果材料不足，Skill 可以参考外部简历/JD 匹配项目的机制，例如：

- 如何抽取 JD 关键词。
- 如何做简历匹配度评分。
- 如何判断 ATS 关键词覆盖。
- 如何评分 bullet 质量。
- 如何区分事实、推断和待补数据。

外部参考是“改写机制兜底”，不是去网上抄简历。

## 输出内容

- 岗位速读：一句话说明 JD 最想要什么人。
- 匹配度判断：当前经历能不能投、优势和风险是什么。
- JD 关键词地图：已覆盖和待补关键词。
- 经历证据库存：用户事实、合理转译、待确认补充分开列。
- 改写策略：每段经历往哪个能力点靠。
- 可切入岗位与写法：告诉用户这段经历可以往哪些岗位靠，每个岗位应该抓什么问题、动作和指标。
- 直接可复制改写版本：给出可以直接放进简历的最终文本，保留必要的数据占位。
- 一页简历排序建议：哪些经历前置，哪些弱化。
- 素材补齐清单：还需要补哪些真实数据和项目细节。
- 面试复用提示：后续面试可能怎么被追问。

## 第一次使用

你只需要给三类信息，越完整效果越好：

1. 目标岗位 JD：截图、链接、复制文本都可以。
2. 原始经历：简历、项目描述、社团经历、实习经历、自我介绍都可以。
3. 输出偏好：飞书文档、本地 Markdown，或先在对话里预览。飞书可用时会用 `lark-cli docs +create --api-version v2` 直接创建 XML 飞书文档；Markdown 只作为飞书不可用时的兜底。

## 推荐输入模板

```text
请使用 resume-jd-tailor，基于下面的 JD 和我的原始经历生成一份岗位定制简历改写文档。

岗位/公司：
JD：
我的原始经历：
目标方向：产品/运营/市场/商业化/未知
输出方式：飞书文档 / 本地 Markdown / 先预览

要求：
1. 先判断岗位匹配度和可投方向
2. 把经历拆成问题、目标、动作、数据
3. 输出可直接放进简历的 bullet
4. 标注哪些数据是待确认，不能替我编经历
5. 给出后续面试可能追问
```

## 和面试 Skill 的配合

这个 Skill 负责把经历改成“能投递的简历证据”。

`interview-prep-brief` 负责把这份简历和 JD 继续拆成“能练习的面试备书”。

推荐链路：

```text
JD + 原始经历 -> resume-jd-tailor -> 定制简历 -> interview-prep-brief -> 面试备书
```
