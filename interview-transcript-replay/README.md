# interview-transcript-replay

面试录音复盘生成器。

它适合用在这样的场景：你已经面完一场面试，手里有录音转写、PDF、飞书文档或纯文本记录，想知道面试官到底问了什么、自己哪里答差了、下次同类问题应该怎么答。

## 它解决什么问题

普通 AI 很容易把面试录音整理成会议纪要，但求职者真正需要的不是“这场面试发生了什么”，而是“下一场怎么答得更好”。

这个 Skill 更强调三件事：

1. 从真实录音里还原面试官问题和追问，而不是泛泛生成题库。
2. 诊断原回答为什么失分，例如太理论、没数据、没答到点、暴露风险。
3. 直接重写成下一次能开口练的答案，并给出追问预判和练习计划。

## 一个重要机制

默认优先使用用户提供的面试录音转写、PDF、飞书文档、JD、简历和项目经历。

如果材料不足以支撑高质量复盘，或者用户明确要求进一步调研，Skill 可以参考 GitHub 上的 mock interview / interview coach / interview feedback / interview transcript analysis 项目。但它参考的不是题目内容，而是产品机制，例如：

- 如何保存 transcript 并基于真实回答做反馈。
- 如何抽取问题、追问和面试官关注点。
- 如何区分 relevance、depth、clarity、communication 等评分维度。
- 如何生成 improvement plan 和 next practice plan。
- 如何追踪多次面试中的重复弱点。

换句话说，外部参考是一个“复盘结构兜底机制”：让面试复盘更可训练，而不是去网上照搬题目。

## 输出内容

- 这场面试暴露了什么：用少量结论说明最大失分点。
- 真实问题地图：还原面试官真实问题、考察点和原回答问题。
- 问题复盘：把真实面试题、原回答风险、下次回答思路、可直接开口答案和追问预判放在一个模块里。
- 下次面试补充材料：用户还需要补的项目、数据、业务知识和指标口径。
- 3 天练习计划：把复盘结果转成可执行练习动作。

## 问题复盘原则

用户文档里只保留一个 `问题复盘` 模块，不再拆成 `核心问题复盘` 和 `其他问题复盘`。用户不需要理解内部分类，只需要快速看到每道题下次怎么答。

内部排序可以按优先级处理：

| 优先级 | 放什么问题 | 目的 |
|---|---|---|
| P0/P1 | 专业能力、业务判断、项目深挖、关键追问 | 修复最影响面试结果的回答 |
| P1/P2 | 自我介绍、离职原因、自我评价、薪资预期、职业规划 | 降低 HR 和综合判断风险 |

不要单独拆出 `HR 问题风险修正` 章节。HR 类问题统一放进 `问题复盘`，这样文档更简洁，也更符合用户复盘习惯。

## 输出方式

优先支持飞书文档；如果当前环境没有配置 `lark-cli`、没有授权或没有飞书权限，则自动生成本地 Markdown 文档。

推荐默认输出：

- 有飞书：`<公司><岗位> 面试录音复盘与下次可用答案` 飞书文档。
- 无飞书：`interview-replays/<公司或岗位>-面试录音复盘.md`。

## 第一次使用

你只需要给几类信息，越完整效果越好：

1. 面试录音转写：PDF、飞书文档、Markdown、纯文本或截图都可以。
2. 岗位信息：公司、岗位 JD、面试阶段。
3. 个人背景：简历、项目经历、自我介绍或几段关键经历。
4. 输出偏好：飞书文档、本地 Markdown，或先在对话里预览。

如果你只有录音转写，Skill 也可以先基于转写完成复盘；但如果补充 JD 和简历，答案会更贴近岗位和个人经历。

## 推荐输入模板

```text
请使用 interview-transcript-replay，基于下面的面试录音转写生成一份复盘文档。

公司/岗位：
面试阶段：一面/二面/终面/HR 面/未知
JD：
我的经历：
面试录音转写：
输出方式：飞书文档 / 本地 Markdown / 先预览

要求：
1. 从录音里提炼真实面试问题和追问
2. 诊断我原回答的问题
3. 统一放在问题复盘模块里输出
4. 给出下次可直接开口的答案和追问预判
5. 最后列出我下次面试前要补的材料和练习计划
```

## 直接答案要求

可直接开口答案不能只把原回答润色一遍，必须包含：

- 对问题的直接结论。
- 原回答缺失的业务动作、判断顺序或指标口径。
- 2-4 个可支撑回答的数据、比例、阈值、时间范围、预算拆分、观察窗口或止损线。
- 面试官可能继续追问的方向。
- 对 HR 风险问题的安全表达。

补充材料和练习计划也要围绕数据缺口展开。不要只写“补齐行业知识”，而要写成用户能准备的材料，例如指标口径表、投放排查表、转化漏斗表、首播检查表，以及每天练习时要达到的量化标准。

如果用户已经提供真实经历，答案应直接基于这些经历写，不要输出 `替换字段`、`替换清单` 或 `示例可替换`。

## 适合谁

- 刚面完一场面试的人：想知道自己哪里答差了。
- 需要复盘录音的人：希望把真实面试转成下一场可用答案。
- 求职辅导者：需要从学生真实面试记录里提炼问题和训练材料。
- 面试连续推进的人：希望把上一场的失分点变成下一轮练习重点。
