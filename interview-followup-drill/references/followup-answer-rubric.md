# Interview Follow-up Answer Rubric

用于构建和检查项目深挖训练，不用于对外展示全部过程。

## 统一故事卡

先写清以下字段，再写任何答案。

| 字段 | 必填内容 |
|---|---|
| 业务背景 | 业务、目标用户、为什么此时要做 |
| 候选人角色 | 自己负责的决策与动作；不要把团队成果都写成个人完成 |
| 原始问题 | 现象、基线、影响范围 |
| 目标 | 指标、周期、约束 |
| 诊断 | 数据、访谈、漏斗或竞品等证据 |
| 方案与取舍 | 备选方案、选择理由、风险 |
| 执行 | 自己做的关键动作、协作方式 |
| 结果 | 指标变化、统计周期、是否存在副作用 |
| 复盘 | 没做好的地方和下一步 |

## 指标一致性

- 为每个关键指标写明分子、分母、时间范围和基线。
- 回答“提升 20%”时明确是百分点、相对提升还是绝对值变化。
- 如果无法确认数据，保留 `[待确认：指标/周期/口径]`；示例数据只能标为 `示例口径`。
- 同一故事跨问题复用相同数据，不能在后续回答随意增加新成果。

## 追问质量

| 层级 | 追问目标 | 合格回答信号 |
|---|---|---|
| 事实 | 验证亲自参与和项目真实性 | 讲清职责边界、时间线、协作对象 |
| 判断 | 验证问题定义与优先级 | 有证据、能排除替代解释 |
| 方案 | 验证产品/业务取舍 | 说出备选、成本、风险和选择依据 |
| 数据 | 验证指标意识与归因能力 | 解释口径、对照、异常和副作用 |
| 失败 | 验证反思与成长 | 承认不足，说明机制性改进 |
| 迁移 | 验证岗位匹配 | 将方法迁移到目标公司具体场景 |

## 评分标准

| 维度 | 5 分 | 3 分 | 1 分 |
|---|---|---|---|
| 真实性 | 角色、动作、数据都清楚，可继续追问 | 有项目但角色模糊 | 空泛套话 |
| 结构 | 结论-证据-动作-结果-复盘完整 | 有动作和结果，但逻辑跳跃 | 流水账 |
| 数据 | 口径、基线、周期、归因完整 | 有数字但口径不清 | 无数据或虚构数据 |
| 取舍 | 有备选方案和选择逻辑 | 只说最终方案 | 没有判断过程 |
| 追问韧性 | 能解释质疑、承认边界并复盘 | 能回答部分追问 | 一质疑就换话题 |

## 常见风险

- 把“参与”说成“主导”；把团队结果说成个人结果。
- 为了好看虚构精确数字，随后无法解释来源。
- 每道题换一个故事，导致面试官难以追问也无法建立可信度。
- 只讲方法论，不说明实际动作、样本、周期和验证。
- 把 JD 推演题、相似题迁移题说成真实面经。
